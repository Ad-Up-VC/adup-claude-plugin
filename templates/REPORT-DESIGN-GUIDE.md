# ADUP Report Design Guide

Standards for all client-facing and internal reports. Every report generated by ADUP must follow these rules.

---

## Tone & Voice

### Client-Facing Reports
- **Confident, not arrogant.** "This was a strong week" not "Performance was adequate."
- **Business language, not marketing jargon.** Translate every metric:
  | Don't Say | Say Instead |
  |-----------|-------------|
  | CPA | Cost per customer |
  | ROAS | Return on ad spend (define on first use, then use "return" or "Xx" shorthand) |
  | CTR | Click-through rate (or just "X% of people who saw the ad clicked") |
  | CPM | Cost per 1,000 impressions (or skip — clients rarely care) |
  | Impressions | Times the ad was shown |
  | Reach | People who saw the ad |
  | Frequency | How often each person saw the ad |
  | CBO/ASC | Budget optimization / Advantage+ (or skip the acronym entirely) |
  | Conversion funnel | Customer journey / path to purchase |
  | Attribution | How we track which ad gets credit for a sale |
  | Blended ROAS | Blended return / overall return on ad spend |
  | Learning phase | Optimization period (explain: "the platform needs ~50 conversions to learn who to target") |

- **Frame problems as opportunities.** "We see an opportunity to improve mobile checkout" not "Mobile is broken."
- **Be specific, not vague.** "Increase Google Shopping budget by 15% to capture 90 additional customers" not "Consider increasing budget."
- **Lead with the punchline.** Executive summary first, data second. Never make someone read 3 pages to find out if things are good or bad.

### Internal Reports
- **Direct and unfiltered.** Use technical terms freely. CPA, ROAS, CTR are fine.
- **Flag problems bluntly.** "Facebook prospecting is wasting budget" is fine internally.
- **Include action status.** What proposals were created, approved, rejected.

## Document Structure

### Header
Always include:
```
# [Report Type] — [Client Name]
### [Date Range] | Compared to [Previous Period]
```

### Section Order (Client Reports)

1. **Executive Summary** — 2-3 sentences. Verdict + biggest win + biggest concern + key recommendation. This is the only section many clients will read.

2. **Performance Overview** — One table with ALL platforms + blended row. This is the money table.

3. **What Worked** — 3-5 wins. Each must include: what happened, why it worked, and what to do about it (continue/scale/replicate).

4. **What Needs Improvement** — 3-5 issues. Each must include: what happened, root cause analysis, and proposed solution. Never present a problem without a solution.

5. **Recommendations** — Priority table: Priority | Action | Expected Impact | What's Needed. Every recommendation must be actionable and evidence-based.

6. **Attribution Note** — Mandatory. Explains why platform numbers differ from GA4. Use consistent language every time (see template below).

7. **Talking Points** — Structured for account managers: Lead with / Address proactively / Recommend / If they ask about...

### Section Order (Monthly — additional sections)
Insert between "Performance Overview" and "What Worked":
- Conversion Funnel (GA4)
- Budget Utilization
- Channel Mix / Revenue Attribution
- Creative Performance Highlights

## Table Design

### Formatting Rules
- **Currency:** Always use the client's currency symbol. Round to whole numbers above €100 (€7,241 not €7,241.23). Two decimal places for CPC/CPA under €100 (€12.13).
- **ROAS/Return:** Always use "Xx" format (3.8x not 380%). Bold the return column — it's what clients look at first.
- **Percentages:** One decimal place (12.4% not 12.39%). Always include + or - sign for changes (+8% not 8%).
- **Blended row:** Always bold. Always last row. Always uses GA4 as source.
- **vs Previous column:** Use words not just numbers. "+8% return" is clearer than "+8%". Color coding when possible (green for improvement, red for decline).

### Platform Name Standards
| Raw | Display As |
|-----|-----------|
| facebook | Facebook & Instagram |
| google | Google Ads |
| tiktok | TikTok Ads |
| linkedin | LinkedIn Ads |
| ga4 | Google Analytics / GA4 |
| blended | All Channels (Blended) |

### Benchmark References
Include benchmarks inline when relevant, not as separate tables:
- "Cost per lead of €78 (below the €100 B2B benchmark)"
- "Click-through rate of 1.8% (above the 1-2% good range)"
- "Return of 2.1x (below the 3x e-commerce target)"

## Numbers That Matter

### Always Include
- Total spend (all platforms combined)
- Total revenue (GA4)
- Blended return (GA4 revenue / total spend)
- Cost per customer or cost per lead
- Period-over-period change (WoW or MoM with %)

### Include When Relevant
- Conversion funnel drop-off rates
- Budget utilization
- Creative lifespan / fatigue status
- Impression share (Google Ads scaling opportunity)
- Hook rate (TikTok)
- Lead quality metrics (LinkedIn → CRM pipeline)

### Never Include (in client reports)
- Raw impression counts without context
- CPM (unless client specifically asks)
- Quality Score details (translate to "ad relevance improved")
- Platform-specific technical jargon (auction mechanics, learning phase internals)
- Micro values from Google Ads (always divide by 1,000,000 first)

## Attribution Note — Standard Text

Use this consistently in every report that mixes platform and GA4 data:

> **How we measure:** Google Analytics 4 (last-click attribution) is the single source of truth for all revenue and customer numbers in this report. Each advertising platform counts conversions using its own method — Facebook includes people who saw an ad but didn't click, while Google distributes credit across multiple touchpoints. These different methods will always produce higher numbers than Google Analytics. This is normal and expected. We use platform numbers for optimizing within each channel, and Google Analytics for cross-channel budget decisions and total revenue reporting.

## PPTX Slide Design Rules

### Slide Structure

**Weekly Report (7 slides):**
1. Title — Client name, date range, one-line verdict
2. Executive Summary — 3-4 bullets, spend + return + key insight
3. Performance Overview — Platform table with blended row
4. What Worked — Top 3 wins, one bullet each
5. What Needs Attention — Top 2-3 items with proposed solutions
6. Recommendations — Priority table
7. Next Steps — Talking points for discussion

**Monthly Report (10-11 slides):**
1. Title
2. Executive Summary
3. Platform Performance
4. Conversion Funnel
5. Budget Utilization
6. Channel Mix / Revenue Attribution
7. Creative Performance Highlights
8. What Worked (top 5)
9. What Needs Improvement (top 5)
10. Strategic Recommendations
11. Next Steps

### Slide Content Rules
- **Maximum 5 bullet points per slide.** If you need more, split into two slides.
- **Every table must fit on one slide.** If too wide, abbreviate column headers.
- **One chart/table per slide.** Never combine chart + table on one slide.
- **Bottom of every slide:** Small text — "Source: ADUP Performance Marketing | [Date]"
- **Title slide must include the verdict.** Not just "Monthly Report" but "Monthly Report — Strong Month, Return Up 9%"

## Quality Checklist — Run Before Saving

Before finalizing any report, verify:

- [ ] Executive summary gives a clear verdict in the first sentence
- [ ] Blended ROAS uses GA4 revenue (not platform-reported)
- [ ] Google Ads values divided by 1,000,000 (no raw micros)
- [ ] All currency values use the correct symbol (€, $, £)
- [ ] ROAS displayed as "Xx" format
- [ ] Period comparison included with +/- percentage
- [ ] Every problem has a proposed solution
- [ ] Every recommendation has an expected impact
- [ ] Attribution note included
- [ ] Talking points section included (client reports)
- [ ] No marketing jargon without translation (client reports)
- [ ] Learning phase entities flagged but not judged
- [ ] Tables have a bold blended/total row
- [ ] File saved with correct naming convention
- [ ] PPTX generated (client-facing reports only)
